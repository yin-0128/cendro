# Datasets

DPO preference data: each row is `{prompt, chosen, rejected}` where `prompt` is the exact
Cendro review request (see [`api/prompts.py`](../api/prompts.py)), `chosen` is a specific,
opinionated review with a concrete fix, and `rejected` is a generic, low-signal review.

| File | Pairs | Source | Committed? |
|------|------:|--------|:----------:|
| `seed_pairs.jsonl` | 12 | Hand-written | ✅ |
| `curated_pairs.jsonl` | 12 | Generated by a local **qwen2.5-coder:14b** judge from `samples/`, then validated | ✅ |
| `train_pairs.jsonl` | 24 | `seed_pairs` + `curated_pairs` merged — **the recommended training input** | ✅ |
| `dpo_pairs.jsonl` | — | Your own generated output | ❌ (git-ignored) |
| `samples/` | 12 files | Reproducible buggy-code corpus used to generate `curated_pairs` | ✅ |
| `raw/` | — | Your own private code to generate from | ❌ (git-ignored) |

## Regenerate / grow the data (free + local)

```bash
python scripts/generate_preferences.py \
  --input dataset/samples/ \
  --provider ollama --model qwen2.5-coder:14b \
  --output dataset/dpo_pairs.jsonl
```

Then validate/curate and (optionally) merge into `train_pairs.jsonl`. The local judge is decent
but not a frontier model — skim the pairs before training. Bigger judge (`:32b`) = better pairs.

## Train on it

```bash
python model/dpo_train.py --config configs/qlora_7b.yaml \
  --dataset dataset/train_pairs.jsonl --output model/output/cendro-7b
```
